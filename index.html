jQuery.slide-area creates any kind of sliding areas. Allows to implement:

  • Any number of pickers
  • Multiple picker values (like x, y coords)
  • Different picker behaviour
  • Specific shapes: triangle, circular, svg
  • Rigid snapping, fluid snapping
  • And more

Picker options

You ask

Why not jquery-ui slider?
At first, it’s jquery ui, what means a ton of unnecessary code. At second, it means jquery, which is known as not the very fast thing. jquery.slide-area, despite of it’s name, fully buit on plain JS, and hooks up jquery plugin only if jquery/zepto is on, but work as well without jquery nonetheless.
Why not any other stand-alone solution?
Well, there’s not that much of sliders, providing that extensibility and generalized approach.
Why not native input[type=range]?
At first, native input doesn’t allow to spawn any number of pickers you need. At second, native input doesn’t measure alternative value (like second axis) of picker. At third, native input doesn’t allow to define custom behaviour of pickers, like repeating. For the rest, there’s a bunch of useful options.
